Example of a delimited input filter

If the input data file (with a filename Great River_Waterside_Q.csv) looks like the following: 

Station·name:·Waterside,·Great·River
Parameter:·River·flow
Date/time,value,quality·flag
2008/01/01·00:00:00,1.234,G
2008/01/01·00:15:00,2.634,G
2008/01/01·00:30:00,,M
2008/01/01·00:45:00,1.738,G

Note that the data/time, value and quality flag are separated by a specific character, a "comma" in this case. We only need to count the position of the year, month, day, hour and minute characters within the date/time column. Count the position of the characters within the column from left to right ("·" indicates a "space") to get the correct character positions for the filter configuration.

The filter file should be set up as follows: 

<?xml version="1.0" encoding="utf-8"?>
<formats>
  <!--
  Start of the filter.
  Other filters may precede.
  --!>
  <format>
    <name>A csv-type filter (*.csv)</name>
    <fileName>
      <extension>csv</extension>
    </fileName>
    <firstDataLine>4</firstDataLine>
    <fileFormat>
      <type>1</type>
      <columnDelimiter>,</columnDelimiter>
    </fileFormat>
    <date>
      <column>1</column>
      <yearStartChar>1</yearStartChar>
      <monthStartChar>6</monthStartChar>
      <dayStartChar>9</dayStartChar>
    </date>
    <time>
      <column>1</column>
      <hourStartChar>12</hourStartChar>
      <minuteStartChar>15</minuteStartChar>
    </time>
    <value>
      <column>2</column>
    </value>
    <qualityFlag>
      <column>3</column>
      <missingValueFlag>M</missingValueFlag>
    </qualityFlag>
  </format>
  <!--
  End of the filter.
  Other filters may follow below.
  --!>
</formats>

Xml-files are case sensitive, the correct case should be used for the node names!

For tab-delimited files use "/t" to specify the tab.
