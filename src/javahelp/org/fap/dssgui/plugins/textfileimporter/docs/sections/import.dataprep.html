Preparing input data

Before time series data are imported from one or more text files, the data should be prepared as follows:

  • Input data should be available as plain text files.
  • Data for all locations (e.g. flow monitoring stations) should be in separate files. Multiple files per station are, however, allowed (this data will be imported as a single dataset).
  • Each data file must have at least a column with date/time values and a column with parameter values. A quality flag column is optional.
  • The time interval of the input data should be constant, e.g. 15mins. All missing records should be represented by a line/row within the input data file.
  • The formatting of the input data files should correspond to one of the import filters. Otherwise a new import filter should be defined. Read more on creating new import filters.

  • Each input data file name should be set as follows:

    Watershed_Location_Other(optional)parts_Parameter.extension

    For example:

    Clyde_Daldowie_2008_Q.txt
    Water of Leith_Murrayfield_H.zrx

    The Parameter part can be specified according to the table below:

    Parameter Specify as
    Flow Q
    Stage H
    Precipitation P
    Temperature T

Data are saved into the HEC-DSSVue database according to the watershed, location and parameter specified in the file names. Note that existing data will be overwritten if there is an overlap in time periods.
