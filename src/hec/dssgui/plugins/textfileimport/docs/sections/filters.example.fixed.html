Example of a fixed-width input filter

If the input data file (with a filename Great River_Waterside_Q.txt) looks like the following: 

Station·name:·Waterside,·Great·River
Parameter:·River·flow
Time·interval:·15mins
Date/time·value·quality·flag
2008/01/01·00:00:00····9.234·G
2008/01/01·00:15:00···12.634·G
2008/01/01·00:30:00··········M
2008/01/01·00:45:00····8.738·G

Note that the data/time, value and quality flag are not separated by a specific character. Instead, each "column" has a fixed number of charachters, including spaces. Count the position of the characters on a line from left to right ("·" indicates a "space") to get the correct character positions for the filter configuration.

The filter file should be set up as follows: 

<?xml version="1.0" encoding="utf-8"?>
<formats>
  <!--
  Start of the filter.
  Other filters may precede.
  --!>
  <format>
    <name>Some filter (*.txt)</name>
    <fileName>
      <extension>txt</extension>
    </fileName>
    <firstDataLine>5</firstDataLine>
    <fileFormat>
      <type>0</type>
    </fileFormat>
    <date>
      <yearStartChar>1</yearStartChar>
      <monthStartChar>6</monthStartChar>
      <dayStartChar>9</dayStartChar>
    </date>
    <time>
      <hourStartChar>12</hourStartChar>
      <minuteStartChar>15</minuteStartChar>
    </time>
    <value>
      <startChar>18</startChar>
      <endChar>26</endChar>
    </value>
    <qualityFlag>
      <startChar>27</startChar>
      <endChar>99</endChar>
      <missingValueFlag>M</missingValueFlag>
    </qualityFlag>
  </format>
  <!--
  End of the filter.
  Other filters may follow below.
  --!>
</formats>

Xml-files are case sensitive, the correct case should be used for the node names!
